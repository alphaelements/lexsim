# --- 漢字 + 々 (踊り字) — 補完文は不要と判明したため追加しない ---
# 々 (U+3005) は従来 tokenize の非分かち書きスクリプト範囲にも segmenter の
# 漢字クラスにも含まれず、スクリプト分割の段階で 人々 → 人/々 に割れていた
# (モデル以前の問題)。コード側で 々 を漢字クラスに編入して再学習したところ、
# seed_corpus.txt の既存 9 例 (人々/島々/半々/年々/月々×2/色々/倍々/清々しかった) だけで
# 漢字|々 連接は非境界と学習され、未知語 (木々/山々/隅々/佐々木/代々木 等)
# にも汎化した。補完文の追加はむしろ悪化を招いた:
#   27 例 → 境界 precision 0.8914 → 0.8850
#   10 例 → word F1 0.8069 → 0.7985 (ゲート 0.80 割れ)
#    6 例 (々|漢字 境界狙い) → 境界 F1 0.9045 → 0.9036、時々雨 の結合も解消せず
# (可愛い の件と同じ「少数例の過剰投入」パターン)。
# 既知の制約: 々 語直後に漢字語が続くと結合する (時々雨/年々増加 が 1 トークン)。
# 分割前 (時/々/雨) と違い 々 単独トークンは漏れず、語幹は保たれるため許容。
# 挙動は tests/segmenter_quality.rs の iteration_mark_* テストで固定している。

# === 2. 技術ドキュメント文体 (説明文、専門用語複合語) ===
形態素 解析器 は 辞書 に 依存 する が 、 lexsim は 辞書 不要 だ 。
この MCP サーバー は Rust で 書かれており 、 辞書 レス の トークナイザー を 使っている 。
lexsim は bigram 分割 な ので 、 メモリ という 単語 が メモ と モリ に 割れて しまう 。
BM25 スコア で 重要 な キーワード を 抽出 し たい が 、 助詞 や 接続語 が 邪魔 に なっている 。
Jaccard 係数 は 2つ の 集合 の 類似度 を 測る ため の 統計 量 だ 。
tfidf_keywords 関数 は 、 コーパス 全体 から TF-IDF 上位 の キーワード を 抽出 する 。
corpus_diff 関数 は 、 2つ の コーパス 間 の 差分 語 を 集合 演算 で 求める 。
NFKC 正規化 を 行う こと で 、 全角 と 半角 の 表記 揺れ を 統一 できる 。
UAX#29 は Unicode の テキスト 分割 標準 で 、 英数字 の 単語 境界 判定 に 使わ れる 。
AdaBoost は 弱 学習 器 を 逐次 加算 する ブースティング アルゴリズム だ 。
snake_case や camelCase の 識別子 は 、 サブ トークン に 分解 して も 検索 できる よう に する 。
辞書フリー な トークナイザー は 、 未知 語 に も 頑健 に 対応 できる 利点 が ある 。
文字 種 分類 は 漢字 、 ひらがな 、 カタカナ 、 英字 、 数字 、 記号 に 大別 さ れる 。
TF-IDF は 文書 中 の 単語 の 重要 度 を 測る 指標 で 、 頻度 と 希少 性 を 組み合わせる 。
コサイン 類似度 は ベクトル 空間 モデル で 広く 使わ れる 距離 指標 の 一つ だ 。
n-gram 特徴 量 は 、 隣接 する 文字 や 単語 の 組み合わせ を 特徴 として 扱う 手法 だ 。
バイナリ 形式 の モデル ファイル は include_bytes! マクロ で 実行 ファイル に 焼き込ま れる 。
FNV-1a は 高速 な 非 暗号 学的 ハッシュ 関数 で 、 特徴 文字列 の ハッシュ 化 に 使わ れる 。
形態素 解析 の 精度 は 、 学習 コーパス の 質 と 量 に 大きく 左右 さ れる 。
トークナイザー の 出力 は 、 下流 の キーワード 抽出 や 類似度 計算 の 入力 として 使わ れる 。
辞書 不要 の アプローチ は 、 メンテナンス コスト を 大幅 に 削減 できる 一方 、 精度 面 で は 課題 が 残る 。
文字 bigram は 、 隣接 する 2 文字 を 一つ の トークン として 扱う 手法 だ 。

特徴 テンプレート に は 、 単一 文字 、 文字 bigram 、 文字 種 の 組み合わせ が 含ま れる 。
ストップワード フィルタ は 、 助詞 や 助動詞 など の 機能 語 を 除去 する 処理 だ 。

感情 分析 は 、 辞書 ベース の 手法 と 機械 学習 ベース の 手法 に 大別 さ れる 。
